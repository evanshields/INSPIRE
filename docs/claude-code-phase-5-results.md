### Critical Issues

#### CRIT-P5-001: Missing `aria-describedby` on Form Fields
- **Task:** Multiple form sections
- **Category:** Accessibility
- **Priority:** Critical (WCAG 2.1 AA Compliance)
- **Description:** Many form fields are missing `aria-describedby` linking to help text or error messages
- **Impact:** Screen reader users may not receive important field instructions or error messages
- **Steps to Reproduce:**
  1. Search code for input fields with help text
  2. Verify if `aria-describedby` links input to help text
  3. Check error message implementation

- **Expected Behavior:** All inputs with help text should have:
  ```html
  <input id="field-name" aria-describedby="field-name-help" />
  <p id="field-name-help">Help text here</p>
  ```

- **Actual Behavior:** Many fields have help text but no `aria-describedby` attribute

- **Example:**
  ```html
  <!-- CURRENT (Missing aria-describedby) -->
  <input type="text" id="property-address" name="property.address" required />
  <p class="text-xs text-gray-500 mt-1">Enter the full property address</p>

  <!-- SHOULD BE -->
  ```
